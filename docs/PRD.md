## 1. 产品概述

### 1.1 产品背景

随着A股市场投资者数量的增加，普通投资者面临着信息不对称、分析工具缺失等问题。专业的量化分析工具通常价格昂贵或使用门槛高，普通投资者难以获取有效的投资决策支持。

InStock旨在为个人投资者提供一个**免费、开源、易用**的智能股票分析平台，通过自动化的数据采集、技术分析和策略选股，帮助投资者做出更明智的投资决策。

### 1.2 产品定位

- **目标用户**: 个人投资者、量化交易初学者、股票分析爱好者
- **核心价值**: 数据驱动、智能分析、策略选股、回测验证
- **差异化优势**: 
  - 完全开源免费
  - 本地化部署，数据安全可控
  - 中文界面，符合国内投资者习惯
  - 自动化定时任务，无需人工干预

### 1.3 产品目标

| 目标类型 | 具体目标 |
|---------|---------|
| 数据目标 | 每日自动更新5000+股票数据，支持60天历史数据查询 |
| 功能目标 | 提供10种选股策略，支持32种技术指标，61种K线形态识别 |
| 性能目标 | API响应时间 < 500ms，前端首屏加载 < 3s |
| 用户体验 | 支持PC和移动端，响应式设计 |

## 2. 用户角色与场景

### 2.1 用户角色

| 角色 | 描述 | 核心需求 |
|------|------|---------|
| 普通投资者 | 非专业个人投资者，希望获取投资参考 | 简单易用，直观展示，明确的买卖信号 |
| 量化爱好者 | 对量化交易感兴趣的学习者 | 策略源码参考，回测功能，技术指标 |
| 数据分析师 | 需要股票数据的分析人员 | 数据导出，API接口，历史数据查询 |

### 2.2 用户场景

**场景一：日常选股**

> 张先生是一名上班族，每天下班后想快速了解今日市场热点。他打开InStock平台，在"策略选股"页面查看今日策略选股结果，发现"放量上涨"策略选中了10只股票。他点击其中一只查看详情，分析K线图和技术指标后决定是否关注。

**场景二：技术分析**

> 李女士是一名技术分析爱好者，她发现某只股票近期表现不错，想深入了解其技术形态。她在"股票详情"页面查看该股票的K线图、技术指标和形态识别结果，综合判断后制定交易计划。

**场景三：策略回测**

> 王先生开发了一个新的选股策略，想验证其历史表现。他使用"回测"功能，设置策略参数和时间范围，运行回测后查看收益率、最大回撤、夏普比率等指标，评估策略有效性。

## 7. 约束条件

### 7.1 技术约束

- 后端使用Python 3.11+，FastAPI框架
- 前端使用Vue 3，TypeScript
- 数据库使用PostgreSQL + TimescaleDB
- 必须支持Docker容器化部署

### 7.2 业务约束

- 仅支持A股市场
- 不提供实盘交易功能
- 数据仅供参考，不构成投资建议

### 7.3 法律约束

- 数据来源于公开渠道
- 遵守相关数据使用协议
- 用户需自行承担投资风险

| RSI | 相对强弱指数，震荡类指标 |
| 龙虎榜 | 交易所公布的异动股票交易明细 |
| 资金流向 | 主力、散户等资金买卖情况 |
